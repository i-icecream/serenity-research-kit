Serenity Research Kit

一个个人版投研技能包：先判断标的是不是供应链卡点，再用买方 memo、增长估值、TAM-PEG、趋势健康度和新闻转 alpha 模块做复核。

来源：muxuuu/serenity-skill 来源：haskaomni/serenity-skill 用途：标的分析 + 估值判断
安装方式 Copy skills/* to ~/.codex/skills/

技能清单

默认从 `serenity-stock-research` 进入；其他 skill 只在问题需要时调用。

serenity-stock-research

入口
什么时候用
你只说“分析某个标的”“做完整判断”“卡点 + 估值一起看”时。
怎么用
让它先路由：卡点判断走 bottleneck，估值走 memo/growth/TAM，择时健康度走 DMA。
原理
不把所有框架混成一锅，而是按问题类型调用最合适的子模块。
Use $serenity-stock-research to analyze [ticker]. Start with the supply-chain bottleneck thesis, then run valuation and bull/base/bear checks.

serenity-bottleneck-research

卡点
什么时候用
判断公司是不是 AI/半导体/硬件/材料链上的真实瓶颈。
怎么用
要求输出产业链位置、稀缺层级、证据强弱、红旗、证伪条件。
原理
从下游需求沿供应链往上游追，找低替代、长认证、扩产慢、买方不得不付钱的层。
Use $serenity-bottleneck-research to judge whether [company] is a real supply-chain bottleneck. Show the layer, evidence, risks, and falsification conditions.

serenity-equity-memo

Memo
什么时候用
需要完整单股研究、投资委员会 memo、SOTP、bull/base/bear 情景。
怎么用
给 ticker，要求拉最新 filings/IR/电话会，形成投资观点、估值、催化剂和监控表。
原理
买方研究先回答“市场哪里错了”，再用财务、估值和证伪点检验。
Use $serenity-equity-memo to create a buy-side memo for [ticker], including financials, valuation scenarios, catalysts, risks, and monitoring metrics.

serenity-growth-valuation

增长估值
什么时候用
争议点是“市场给的估值是否已经反映未来 3-5 年增长”。
怎么用
让它拆 H0-H5 增长假设，比较内在增长、市场隐含增长和 FOMO 溢价。
原理
把每条新信息当作对未来增长概率的更新，而不是简单说利好/利空。
Use $serenity-growth-valuation to compare [ticker]'s intrinsic 3-5Y growth with market-implied growth and identify FOMO or multiple-expansion risk.

serenity-tam-peg

TAM-PEG
什么时候用
成长股 PE/PEG 看起来贵，但可能有超长 TAM runway 或高质量卡位。
怎么用
输入 forward PE、EPS CAGR、TAM 增速/渗透率、质量因子和稀释风险。
原理
传统 PEG 只看增长速度；TAM-Adj-PEG 同时看增长能持续多久、能否落到公司利润里。
Use $serenity-tam-peg to evaluate whether [ticker]'s valuation is justified by EPS growth, TAM runway, quality factor, and dilution risk.

serenity-dma-health

健康度
什么时候用
标的涨太多、跌到均线附近、想判断趋势是否被基本面支持时。
怎么用
需要最新价格、20/50/100/200DMA、ATR、指引、预期修正和基本面增速。
原理
比较股价均线速度和基本面速度，判断是健康趋势、过热逃逸，还是基本面变差后的破位。
Use $serenity-dma-health to score [ticker]'s valuation and trend health using fundamentals, moving averages, divergence, and estimate revisions.

serenity-alpha

新闻转 Alpha
什么时候用
起点是一条新闻、产品发布、订单、采购、客户线索或电话会暗示。
怎么用
让它判断新闻是否已经变成可观察需求，再映射到收入、利润、现金流和小市值弹性。
原理
新闻本身不重要，重要的是它能否改变某家小公司未来几个季度的财务报表。
Use $serenity-alpha to analyze this news: [paste news]. Map observable demand, financial transmission, small-cap beneficiaries, risks, and validation metrics.

场景选择

不要每次全开。问题越清楚，调用越少，结论越干净。

你的问题 优先调用 补充调用 最后要拿到什么
“帮我完整分析一个 ticker” serenity-stock-research 自动组合 bottleneck + memo + valuation 卡点真伪、估值区间、催化剂、风险、证伪条件
“它是不是 CPO/AI 供应链卡点” serenity-bottleneck-research serenity-equity-memo 卡在哪层、谁依赖它、证据强弱、替代风险
“这票现在贵不贵” serenity-equity-memo serenity-growth-valuation / serenity-tam-peg 市场隐含预期、估值情景、上行/下行条件
“新闻出来后谁最受益” serenity-alpha serenity-bottleneck-research 需求是否真实、财务传导路径、小公司弹性候选
“涨太多还能不能追，跌下来是否健康” serenity-dma-health serenity-growth-valuation 趋势是否被基本面支持、是否过热、是否破位
“几个候选里谁优先研究” serenity-bottleneck-research serenity-equity-memo 先排产业链层级，再排公司，最后看估值红旗

组合流程

完整单股分析建议用这条链路。它把“像 Serenity 一样找卡点”和“买方估值判断”分开处理。

1. 定义问题ticker、市场、时间窗口、你关心的是卡点、估值还是新闻。
2. 画产业链从下游 capex 或需求开始，上溯到组件、设备、材料、衬底。
3. 判卡点看供应商数量、认证周期、替代方案、扩产难度和客户依赖。
4. 做估值用 memo、增长隐含预期、TAM-PEG 或 SOTP 检查价格是否透支。
5. 看时点用催化剂、财报节点、估计修正和 DMA 健康度判断当前状态。
6. 证伪复核列出什么事实会推翻 thesis，避免只为看多找理由。

常用组合

这些 prompt 可以直接复制改 ticker。

完整标的分析

  1. 先用 `serenity-bottleneck-research` 判断卡点和证据。
  2. 再用 `serenity-equity-memo` 做财务、估值、情景。
  3. 最后用 `serenity-growth-valuation` 检查市场隐含增长。
Use $serenity-stock-research to analyze [ticker]. Start with bottleneck strength, then run buy-side valuation, bull/base/bear scenarios, catalysts, risks, and falsification checks.

新闻转投资线索

  1. 先用 `serenity-alpha` 判断是否有真实需求变化。
  2. 再用 `serenity-bottleneck-research` 找受益链条中的稀缺层。
  3. 只把候选当作待验证清单，不直接当结论。
Use $serenity-alpha and $serenity-bottleneck-research to analyze this news: [news]. Find observable demand, financial transmission, bottleneck layers, candidate companies, and verification metrics.

估值复核

  1. 先用 `serenity-equity-memo` 建 bull/base/bear。
  2. 成长股再加 `serenity-tam-peg`。
  3. 高热度标的再加 `serenity-growth-valuation` 检查 FOMO。
Use $serenity-equity-memo, $serenity-growth-valuation, and $serenity-tam-peg to value [ticker]. Separate reported facts, management claims, assumptions, and pure speculation.

当前位置健康度

  1. 先确认基本面没有被证伪。
  2. 再用 `serenity-dma-health` 看趋势是否过热或破位。
  3. 把它当状态检查，不当独立买卖信号。
Use $serenity-dma-health to evaluate [ticker]'s current trend health. Compare price/DMA divergence with fundamental speed and estimate revisions.

数据源边界

这些 skill 会指导 Codex 去找真实数据和新闻，但 skill 本身不是数据库、行情终端或新闻订阅器。

问题 答案 怎么处理
会不会找真实数据？ 会，只要当前 Codex 环境有联网、浏览器、搜索、行情或文件工具。 skill 会要求优先查公司公告、SEC/交易所文件、IR、财报电话会、新闻和可信行业资料。
会不会自动追踪新闻？ 不会。它不是定时新闻雷达，也不会自己后台更新。 你问“最新”“今天”“近期新闻”或贴一条新闻时，Codex 才会按 skill 的规则去查或分析。
能不能用你另一个帖子追踪项目？ 可以，那个项目负责发现线索，这套 kit 负责把线索变成标的研究和估值判断。 把追踪项目产出的 ticker、新闻、推文摘要或主题丢给 `serenity-stock-research` / `serenity-alpha`。
如果没有实时数据怎么办？ 应该明确标注“未核验”或“初步框架判断”。 不要编价格、市值、订单、客户和估值倍数；列出下一步要查的具体来源。

优先数据源

不同模块看不同证据，越靠近最终结论，越需要一手来源。

公司一手资料

10-K、10-Q、8-K、20-F、6-K、年报、季报、交易所公告、IR presentation、earnings release、电话会 transcript。

市场与估值数据

最新价格、市值、EV、forward PE、EV/S、共识预期、估计修正、同业倍数。需要实时或近期数据源支持。

产业链证据

客户/供应商交叉提及、订单、backlog、认证、扩产、专利、标准、招投标、行业会议、可信行业媒体。

新闻和社交线索

新闻、推文、论坛和 KOL 只作线索。只有被公告、财报、客户/供应商或行业数据验证后，才升级为证据。

A 股资料路径

巨潮资讯、交易所问询函/互动易、临时公告、业绩预告、招投标、环评/能评、专利和海关等公开数据。

技术健康度数据

`serenity-dma-health` 需要价格、20/50/100/200DMA、ATR、指引和预期修正。缺数据时只能输出待补字段。

适用边界

这套 kit 有两个层次：卡点框架有明显偏好，估值框架覆盖更广。

公司类型 卡点框架适用度 估值模块适用度 怎么用才对
小/中市值硬科技、材料、设备、零部件、半导体供应链 高 高 先找产业链卡点，再做估值和证伪。这是本 kit 的最强区。
大市值半导体、云厂、平台型 AI 公司 中 高 卡点框架只用来判断链上地位；真正结论主要靠增长、利润率、估值和资本开支周期。
SaaS、互联网平台、消费品牌、金融公司 低 中到高 不要强行找物理卡点。用 equity memo、growth valuation、TAM-PEG 做普通估值和 thesis 复核。
纯周期、商品、宏观交易、ETF 低到中 中 重点看供需周期、价格曲线、资产负债表和趋势健康度，不要把它包装成单公司卡点。
早期亏损、pre-revenue、二元事件公司 视情况 视情况 可以做 milestone/option-style 分析，但必须降低精度，明确融资、稀释和验证节点。

一句话原则

先判断问题属于哪一类，再选择工具。

卡点框架偏小市值

Serenity 式赔率通常来自小公司控制大链条里的稀缺层。市值越大、覆盖越充分，卡点框架越容易只剩定性价值。

估值框架更通用

买方 memo、内在增长、TAM-PEG 和 DMA 健康度可以分析绝大多数上市公司，只是输入指标和估值尺子要换。

不要硬套

如果公司不是物理供应链卡点，就直说“不适合用卡点框架”，转入普通基本面、增长和估值分析。

底层原理

这套 kit 的核心不是追推文，而是把标的研究拆成几个可复核的问题。

先产业链，后 ticker

先问“哪一层供给最难扩、最难替代”，再问“哪家公司离这一层最近”。这样能避免一上来就被热门股票牵着走。

证据分层

财报、公告、电话会、客户/供应商交叉验证优先；社交媒体和二手报道只当线索。越靠近结论，证据要求越高。

估值看隐含预期

不是只说“贵/便宜”，而是反推当前价格需要兑现什么增长、利润率、TAM 份额和估值倍数。

新闻要落到报表

一个新闻只有能影响收入、毛利、订单、现金流或资产负债表，才可能变成 alpha。否则只是叙事热度。

趋势服从基本面

DMA 健康度不是纯技术分析。它检查价格速度和基本面速度是否匹配，防止把过热当强势，把破位当便宜。

证伪比背书重要

每个 thesis 都要明确什么事实会让它降级或失败：替代路线、稀释、客户流失、扩产超预期、估值已透支。
